## 💡 주요 시나리오
- 사용자 관리: 회원가입, 로그인(JWT), 포인트 충전/조회
- 콘서트 예매: 콘서트/스케줄/좌석 조회 후 예매 진행
- 대기열 관리: 예매시 대기열 진입, 대기 상태/순번 조회
- 예약 관리: 좌석 임시 배정(예약), 예약 확정(결제), 예약 취소
- 결제 관리: 포인트 충전, 포인트 결제, 결제/포인트 내역 조회

### 사니리오 1: 사용자 관리
1. 사용자는 회원가입 화면에서 이메일/비밀번호 등 필수 정보를 입력하고 회원가입을 완료한다.
2. 사용자는 로그인 화면에서 아이디/비밀번호를 입력해 로그인 요청을 보낸다.
3. 시스템은 인증에 성공하면 JWT 토큰을 발급하고, 이후 API 요청에 이 토큰을 사용한다.
4. 사용자는 로그인 상태에서 포인트 충전, 예매, 예약 조회 등 기타 기능을 이용할 수 있다.
5. 인증되지 않은 사용자가 예매/결제/예약 조회 API를 호출하면, 시스템은 401 Unauthorized 또는 권한 없음 응답을 반환한다.(비정상 흐름)

### 시나리오 2: 콘서트 예매
1. 사용자는 콘서트 목록에서 원하는 콘서트를 선택한다.
2. 선택한 콘서트의 스케줄 목록과 각 스케줄별 좌석 현황(잔여 좌석)을 조회한다.
3. 사용자는 특정 스케줄을 선택하고 예매하기 버튼을 클릭한다.
4. 시스템은 해당 스케줄에 대한 대기열 토큰을 발급하고, 사용자를 대기열에 진입시킨다.
5. 사용자의 대기열 상태가 활성(Active) 상태가 되면, 사용자는 좌석 선택 화면으로 이동한다.
6. 사용자는 좌석 배정 화면에서 좌석을 조회하고 원하는 좌석을 선택한다.
7. 선택한 좌석에 대해 임시 배정(예: 5분 TTL)이 설정되고, 다른 사용자는 해당 좌석을 선택할 수 없다.\
   (이후 결제 및 확정은 “예약 관리/결제 관리” 시나리오와 연결)

### 시나리오 3: 결제 관리(포인트 기반 결제)
1. 사용자는 포인트 충전 API를 호출해 원하는 금액만큼 포인트를 충전한다.
2. 결제가 승인되면, 충전된 포인트가 사용자 계정의 포인트 잔액에 즉시 반영된다.
3. 사용자는 포인트 조회 API를 통해 현재 잔액과 충전/사용 내역을 확인할 수 있다.
4. 사용자는 콘서트 좌석 예약시, 결제 수단으로 "포인트 결제"를 선택한다.
5. 시스템은 사용자의 포인트 잔액을 확인한 뒤, 잔액이 충분하면 해당 금액만큼 차감하고 예약을 확정한다.
6. 잔액이 부족하면 결제를 실패시키고 예약은 확정되지 않는다.(비정상 흐름)

### 시나리오 4: 대기열 관리
1. 사용자가 콘서트/스케줄의 예매하기 버튼을 누른다.
2. 시스템은 해당 사용자를 대기열에 등로갛고, 고유한 대기열 토큰을 발급한다.
3. 사용자는 대기열 페이지에서 자신의 대기 순번, 예상 대기 시간, 상태(대기/활성) 등을 조회할 수 있다.
4. 사용자의 순번이 도착해 활성(Active) 상태가 되면, 좌석 선택 화면으로 이동할 수 있는 권한이 생긴다.
5. 일정 시간(예: 5분) 내에 좌석 선택을 시작하지 않으면, 대기열 권한이 만료되어 뒤에 있는 사용자에게 기회가 넘어간다.

### 시나리오 5: 예약 관리
1. 대기열을 통과한 사용자는 좌석 선택 화면에서 원하는 좌석을 선택한다.
2. 사용자가 좌석을 선택하면 해당 좌석에 임시 배정(예: 5분 TTL)이 설정된다.
3. 사용자는 임시 배정 상태에서 결제 화면으로 이동한다.
4. 사용자가 결제 방식을(포인트/카드/계좌이체) 선택하고 결제를 완료하면, 해당 예약은 확정 상태로 변경되고 좌석은 최종 배정(Sold_out) 된다.
5. 사용자가 결제 화면에서 취소를 선택하거나, 임시 배정 시간이 만료되면 좌석 임시 배정이 해제되고 다시 다른 사용자에게 판매 가능 상태가 된다.
6. 사용자는 예약 내역 조회 화면에서 자신의 예약 상태(임시/확정/취소)를 확인 할 수 있다.
